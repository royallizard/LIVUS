Cryptographic engine based on the
Advanced Encryption Standard (AES)
algorithm called Rijndael.
The text is first encrypted with Rijndael
and then converted to %s notation.
The keys used are %d bit long and will be padded
with zeros if you provide shorter ones.
If only one key is provided, this engine
will use it for both encryption and decryption.
This is the Crypto++ implementation which could do
a lot more than the current default engine, but as
this should be a drop-in replacement, it acts just
like the original one.
Information about the algorithm can be found at
<http://www.cryptolounge.org/wiki/AES>

The text is first encrypted with Blowfish 
and then base64 encoded.
Keys of variable length can be used and
are specified as character strings.
Keys can be up to 56 bytes (448 bits) long.
If only one key is provided, this engine
will use it for encryption and decryption.
This engine works in ECB mode by default,
if you want to use CBC mode you must prefix
your key(s) with "cbc:".
